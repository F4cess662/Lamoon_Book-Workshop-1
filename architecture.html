📋 องค์ประกอบของระบบ | System Components

👥 บทบาท (Actors)

👤 Customer (ลูกค้า)

ผู้ซื้อหนังสือที่เข้ามาใช้ระบบเพื่อค้นหา สั่งซื้อ และชำระเงิน

👔 Staff (พนักงาน)

บุคลากรของร้านที่จัดการคำสั่งซื้อ ยืนยันการชำระเงิน และอัปเดตสถานะ

🔐 Admin (ผู้ดูแลระบบ)

ผู้บริหารระบบที่จัดการข้อมูลหนังสือ ผู้ใช้งาน และรายงาน

📱 ชั้นการนำเสนอ (Presentation Layer)

Customer Interface - หน้าเว็บสำหรับลูกค้าในการค้นหา สั่งซื้อ และติดตามคำสั่งซื้อ
Staff Dashboard - แผงควบคุมสำหรับพนักงานในการจัดการคำสั่งซื้อและการชำระเงิน
Admin Dashboard - แผงควบคุมสำหรับผู้ดูแลระบบในการจัดการข้อมูลและรายงาน

🔧 ชั้นธุรกิจ (Business Logic Layer)

👤 User Service

จัดการการลงทะเบียน เข้าสู่ระบบ และข้อมูลผู้ใช้งาน

📚 Book Service

จัดการรายการหนังสือ หมวดหมู่ และข้อมูลรายละเอียด

🛒 Order Service

ประมวลผลการสั่งซื้อ การจัดส่ง และติดตามสถานะ

💳 Payment Service

จัดการการชำระเงิน การตรวจสอบ และการยืนยัน

📦 Inventory Service

ติดตามสต็อกสินค้า และจัดการการขาดสินค้า

💾 ชั้นข้อมูล (Data Layer)

Database (MySQL/PostgreSQL) - ฐานข้อมูลสำหรับเก็บข้อมูลทั้งหมดของระบบ
Cache (Session/Data) - หน่วยเก็บข้อมูลชั่วคราวเพื่อเพิ่มความเร็ว

🌐 ระบบภายนอก (External Systems)

Email Service - ระบบส่งอีเมลสำหรับการแจ้งเตือนลูกค้า
Payment Gateway - ประตูทางสำหรับการชำระเงินออนไลน์

🔄 การไหลของข้อมูล | Data Flow

1. Customer ค้นหาหนังสือ

Customer UI → User Service (Authentication) → Book Service → Database

2. Customer สั่งซื้อ

Customer UI → Order Service → Inventory Service → Payment Service

3. Staff จัดการคำสั่งซื้อ

Staff Dashboard → Order Service → Database → Email Service (แจ้งเตือน)

4. Admin ดูรายงาน

Admin Dashboard → User Service / Book Service / Order Service → Database
