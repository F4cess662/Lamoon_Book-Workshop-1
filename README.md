# ระบบร้านขายหนังสือ
# Wholesale Book Store Management System

## 📋 ภาพรวมโครงการ | Project Overview

ระบบร้านขายหนังสือ (Wholesale Book Store Management System) เป็นระบบจัดการเชิงผสมผสาน ที่ได้รับการออกแบบเพื่อบริหารจัดการการขายหนังสือให้กับร้านค้า โรงเรียน ห้องสมุด และลูกค้าองค์กร โดยระบบนี้สามารถจัดการข้อมูลหนังสือ สต็อกสินค้า การสั่งซื้อ และการชำระเงินได้อย่างมีประสิทธิภาพและโปร่งใส

## 🎯 วัตถุประสงค์โครงการ | Objectives

1. **พัฒนาระบบจัดการร้านขายหนังสือแบบขายส่ง** - สร้างแพลตฟอร์มดิจิทัลสำหรับการบริหารจัดการการขายหนังสือที่มีประสิทธิภาพ
2. **เพิ่มความสะดวกในการสั่งซื้อหนังสือจำนวนมาก** - ลดเวลาและความยุ่งยากในกระบวนการสั่งซื้อสำหรับลูกค้าที่มีปริมาณการสั่งซื้อมาก
3. **จัดการสต็อกสินค้าอย่างเป็นระบบ** - ติดตามสินค้าคงเหลือแบบรีอลไทม์ และแจ้งเตือนเมื่อสินค้าหมด
4. **ลดข้อผิดพลาดในการจัดการคำสั่งซื้อ** - ลดการบันทึกข้อมูลที่ผิดพลาดและเพิ่มความแม่นยำของระบบ
5. **เพิ่มประสิทธิภาพในการบริหารร้านค้า** - เพิ่มผลผลิตและลดเวลาในการประมวลผลข้อมูล

## 📊 ขอบเขตงาน | Scope

### 🛍️ ฝั่งลูกค้า (Customer)

* สมัครสมาชิกและสร้างบัญชีผู้ใช้งาน
* เข้าสู่ระบบด้วยชื่อผู้ใช้และรหัสผ่าน
* ค้นหาหนังสือตามชื่อ หมวดหมู่ หรือผู้แต่ง
* ดูรายละเอียดหนังสือ (ราคา สำนักพิมพ์ คำอธิบาย)
* สั่งซื้อหนังสือจำนวนมากพร้อมการเลือกปริมาณ
* ชำระเงินผ่านหลายช่องทาง
* ตรวจสอบสถานะคำสั่งซื้อและการจัดส่ง

### 👔 ฝั่งพนักงาน (Staff)

* เข้าสู่ระบบพื้นที่จัดการพนักงาน
* ดูและจัดการคำสั่งซื้อ (ยืนยัน/ปฏิเสธ)
* ตรวจสอบการชำระเงินของลูกค้า
* อัปเดตสถานะการจัดส่ง (กำลังเตรียม/อยู่ระหว่างจัดส่ง/จัดส่งเสร็จแล้ว)
* ติดตามสต็อกสินค้าและรายงานการขาดสต็อก

### 🔐 ฝั่งผู้ดูแลระบบ (Admin)

* เข้าสู่ระบบพื้นที่ admin
* เพิ่ม/แก้ไข/ลบข้อมูลหนังสือ
* จัดการหมวดหมู่และหมวดย่อย
* บัญชีผู้ใช้งาน (ลบ/ปิดใช้งาน)
* ดูรายงานยอดขายและการวิเคราะห์ข้อมูล

## 🔍 ปัญหาและความต้องการ | Problem Statement & Pain Points

### ปัญหาเชิงทั่วไป
ร้านขายหนังสือแบบขายส่งส่วนใหญ่ยังคงใช้วิธีการจดบันทึกด้วยตนเองหรือระบบที่ไม่เป็นมาตรฐาน ซึ่งทำให้เกิดปัญหาต่างๆ ในการตรวจสอบสต็อก การจัดการคำสั่งซื้อ และการติดตามการชำระเงิน ส่งผลให้เกิดความผิดพลาดและใช้เวลาทำงานมากขึ้น

### ปัญหาฝั่งลูกค้า (Customer Pain Points)
- ❌ ค้นหาหนังสือและตรวจสอบความพร้อมของสินค้าได้ยาก
- ❌ ไม่ทราบข้อมูลสต็อกสินค้าคงเหลือแบบรีอลไทม์
- ❌ ตรวจสอบสถานะคำสั่งซื้อไม่สะดวกและไม่มีการแจ้งเตือน
- ❌ การสั่งซื้อจำนวนมากใช้เวลานานและต้องติดต่อโดยตรง
- ❌ ไม่มีประวัติการซื้อ (Order History)

### ปัญหาฝั่งร้านค้า (Staff Pain Points)
- ❌ ต้องตรวจสอบสต็อกด้วยตนเองโดยไปตรวจสินค้าจริง
- ❌ เกิดข้อผิดพลาดในการบันทึกคำสั่งซื้อและจำนวนสินค้า
- ❌ ติดตามการชำระเงินจากลูกค้ายาก บ่อยครั้งไม่ชำระหรือชำระไม่ครบ
- ❌ ไม่มีรายงานสรุปยอดขายแบบอัตโนมัติ
- ❌ การจัดส่งอาจสับสนและไม่มีการแจ้งเตือนให้ลูกค้า
